Landing pages estáticas em Astro não têm servidor backend. Mas isso não significa que você não pode processar formulários. Webhooks conectam seu frontend a qualquer serviço sem precisar de um servidor próprio.

## Como Funciona um Webhook em Landing Pages

Quando o usuário envia um formulário, o JavaScript captura os dados e faz uma requisição POST para uma URL de webhook. Essa URL aponta para um serviço como n8n, Make ou Zapier, que processa os dados e os encaminha.

O processo é totalmente serverless: você paga apenas pelo uso e não gerencia infraestrutura.

## n8n: Automação Open-Source

n8n é uma ferramenta de automação open-source que você pode hospedar no seu próprio servidor ou usar a versão cloud. Ela oferece centenas de integrações: CRMs, email marketing, bancos de dados, APIs e mais.

Para landing pages, crie um workflow que:

1. Recebe o webhook com os dados do formulário
2. Valida o payload e verifica honeypot anti-spam
3. Adiciona o lead ao CRM (HubSpot, RD Station, Pipedrive)
4. Envia email de confirmação para o lead
5. Notifica a equipe de vendas no Slack ou WhatsApp

## Make (ex-Integromat)

Make é uma alternativa visual com interface mais amigável que n8n. Ele oferece cenários complexos com roteamento condicional, agregação de dados e transformação de texto.

Uma vantagem do Make é o suporte nativo a webhooks com geradores de schema automáticos. Cole a URL do webhook no seu formulário e o Make mapeia os campos automaticamente.

## Honeypot Anti-Spam

Todo formulário com webhook precisa de proteção contra spam. A técnica mais simples e eficaz é o honeypot: um campo invisível que bots preenchem mas humanos não veem.

Implemente um campo oculto no formulário. Se ele vier preenchido, descarte o envio silenciosamente.

## Payload JSON Estruturado

Estruture o payload do webhook de forma consistente:

```json
{
  "name": "João Silva",
  "email": "joao@email.com",
  "phone": "(11) 99999-9999",
  "source": "landing-page-seo",
  "timestamp": "2026-06-24T10:30:00Z",
  "consent": true
}
```

Use o campo source para identificar qual landing page gerou o lead. Isso permite atribuição correta nos relatórios do CRM.

## Tratamento de Erros

Webhooks podem falhar. Implemente retry automático com backoff exponencial. Serviços como n8n e Make já têm retry nativo. No frontend, mostre uma mensagem de sucesso mesmo se o webhook falhar, e processe em fila.

## Conclusão

Webhooks com n8n e Make eliminam a necessidade de backend para landing pages. Você conecta formulários a qualquer serviço em minutos, sem servidor, sem banco e sem complexidade. A automação que antes exigia um time de desenvolvimento agora está ao alcance de um workflow.
